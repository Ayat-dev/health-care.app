 CLAUDE CODE — WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Installation :
    npm install -g @anthropic/claude-code

  Lancer une session :
    cd medical-app          ← toujours à la racine (là où est CLAUDE.md)
    claude

  CLAUDE.md est lu automatiquement → contexte complet dès le démarrage.

  Exemples d'instructions efficaces :
    "Implémente le module Pharmacie selon docs/modules/05-PHARMACY.md"
    "Ajoute la pagination à la liste patients dans PatientWebController"
    "Corrige le bug dans AppointmentService, la méthode findByDate"
    "Dans le client JavaFX, crée l'écran consultations qui appelle GET /api/consultations"

  AVANT chaque session importante :
    git add . && git commit -m "checkpoint avant session Claude Code"

  Pour chaque session de dev, tu ouvres avec :

"Lis docs/OVERVIEW.md et docs/DATABASE.md, puis implémente le module décrit dans docs/modules/05-PHARMACY.md en respectant les conventions."

 RÈGLES À NE JAMAIS OUBLIER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ✗ Ne jamais modifier une migration Flyway existante → toujours en créer une nouvelle
  ✗ Ne jamais utiliser float/double pour l'argent → BigDecimal
  ✗ Ne jamais hard-delete un dossier médical → soft delete (deleted_at)
  ✗ Ne pas créer de nouveau fichier CSS → étendre app.css
  ✗ Ne pas mettre de logique métier dans les contrôleurs → uniquement dans les services

 CONVENTIONS DE NOMMAGE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Entités          →  PascalCase singulier        ex: StockItem
  DTOs             →  suffixe Dto                 ex: PatientDto
  REST controllers →  suffixe ApiController       ex: PatientApiController
  Web controllers  →  suffixe WebController       ex: PatientWebController
  Services         →  suffixe Service             ex: PatientService
  Migrations       →  V{N}__{description}.sql     ex: V4__pharmacy_tables.sql
  Templates        →  templates/{module}/{vue}.html

 API REST — PATTERN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  GET    /api/{module}                → liste
  GET    /api/{module}/{id}           → un élément
  POST   /api/{module}                → créer
  PUT    /api/{module}/{id}           → modifier
  PATCH  /api/{module}/{id}/{action}  → action partielle (confirm, cancel…)
  DELETE /api/{module}/{id}           → soft delete (ADMIN only)

  Auth JWT : header →  Authorization: Bearer {token}
